# Railway 환경 변수 설정 가이드

## 🚀 배포 상태

✅ **코드가 GitHub에 푸시되었고 Railway가 자동 배포를 시작했습니다!**

## 📝 Railway 환경 변수 설정하기

### 1단계: Railway 대시보드 접속

1. https://railway.app 접속
2. 로그인
3. 프로젝트 선택

### 2단계: 환경 변수 추가

1. **Variables** 탭 클릭
2. **New Variable** 버튼 클릭
3. 다음 환경 변수 추가:

```
HOTPEPPER_API_KEY=ea23188c08fd9123
```

### 3단계: 배포 완료 대기

- 환경 변수를 추가하면 Railway가 자동으로 재배포합니다
- **Deployments** 탭에서 배포 진행 상황 확인
- 초록색 체크 표시가 나타나면 배포 완료!

## 🎉 사용 방법

배포가 완료되면 다음과 같이 사용할 수 있습니다:

### 1. 로그인

Railway 배포 URL로 접속하여 로그인

### 2. 음식점 검색 페이지

사이드바에서 **"リクルート検索"** (일본어) 또는 **"리쿠르트 검색"** (한국어) 클릭

### 3. 검색하기

- **지역 선택**: 도쿄, 오사카, 교토 등
- **키워드 입력**: 라멘, 스시, 이자카야 등
- **검색 및 저장** 버튼 클릭

### 4. 검색 결과

- 자동으로 데이터베이스에 저장됩니다 (최대 100개)
- 중복된 가게는 자동으로 업데이트됩니다

### 5. 저장된 맛집 관리

- 검색 결과 하단에서 저장된 맛집 목록 확인
- 가게명이나 주소로 필터링 가능
- 불필요한 항목은 삭제 가능
- 핫페퍼 사이트 링크로 바로 이동 가능

## 📊 주요 기능

✅ **지역별 검색**: 일본 주요 도시별 음식점 검색
✅ **키워드 검색**: 음식 종류, 가게명으로 검색
✅ **자동 저장**: 검색 결과 자동 데이터베이스 저장
✅ **중복 방지**: 같은 가게는 자동으로 업데이트
✅ **목록 관리**: 저장된 맛집 조회, 검색, 삭제
✅ **외부 링크**: 핫페퍼 사이트로 바로 이동

## 💡 팁

- **한 번에 최대 100개**: API 제한으로 한 번에 100개까지만 수집됩니다
- **일일 제한**: 핫페퍼 API는 일일 호출 제한이 있습니다 (약 3,000~10,000건)
- **복합 검색**: 지역 + 키워드를 함께 사용하면 더 정확한 결과를 얻을 수 있습니다

## 🔧 문제 해결

### 검색이 안 되는 경우

1. Railway Deployments 탭에서 배포가 완료되었는지 확인
2. Variables 탭에서 `HOTPEPPER_API_KEY`가 올바르게 설정되었는지 확인
3. Railway Logs를 확인하여 에러 메시지 확인

### API 에러가 발생하는 경우

- 일일 호출 제한에 도달했을 수 있습니다
- 다음 날까지 기다려주세요

## 📞 지원

문제가 발생하면 Railway Logs를 확인하세요:
- Railway 대시보드 → 프로젝트 선택 → View Logs
